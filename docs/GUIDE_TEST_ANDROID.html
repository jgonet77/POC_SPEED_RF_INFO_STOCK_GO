🚀 Guide de Test - Application Stock

Teste ton app Android sans Android Studio!

0Avant de commencer - Préparation

Tu dois vérifier que tu as tout ce qu'il faut:

  • Un téléphone Android connecté à ton ordinateur par USB
  • OU un émulateur Android lancé
  • Ton serveur Python backend en cours d'exécution
  • Un terminal (cmd.exe ou PowerShell sur Windows)
⚠️ Point important: Tu dois d'abord connaître l'adresse IP et le port du serveur backend!
Exemple: 192.168.1.20:8000 ou localhost:8000 (pour émulateur)

1Ouvrir le terminal et aller dans le dossier du projet

Ouvre le terminal (cmd.exe ou PowerShell) et tape:

cd D:\Projects\POC_SPEED_RF_INFO_STOCK\android

Tu dois voir quelque chose qui ressemble à:

D:\Projects\POC_SPEED_RF_INFO_STOCK\android>
💡 Astuce: Si tu copies le chemin d'un dossier, tu peux faire un clic droit dans le terminal pour coller!

2Compiler l'application Android

Tape cette commande pour créer l'app:

gradlew build

⏳ C'est normal que ça prenne du temps! (2-5 minutes la première fois)

✅ Quand c'est terminé, tu dois voir:
BUILD SUCCESSFUL
❌ Si tu vois une erreur:
- Vérifie que tu es dans le bon dossier (D:\...\android)
- Vérifie que Java est installé
- Relance la commande

3Installer l'app sur ton téléphone ou émulateur

Tape cette commande:

gradlew installDebug

⏳ Ça va prendre quelques secondes...

✅ Quand c'est fait, tu dois voir:
BUILD SUCCESSFUL
Et l'app s'installera automatiquement sur ton téléphone!
❌ Si rien ne se passe:
- Vérifie que ton téléphone est connecté en USB
- Ou que l'émulateur est lancé
- Déverrouille ton téléphone et accepte les permissions

4Lancer l'application

Sur ton téléphone, cherche l'app "Stock App POC" et appuie dessus.

Tu devrais voir cet écran:

🧪 Maintenant... TESTONS L'APP!

⚙️TEST 1: Configurer l'adresse du serveur

  1. Appuie sur le bouton Settings
  2. Tu verras deux champs:
    • API Host (adresse du serveur)
    • API Port (port, normalement 8000)
  3. Efface ce qui est écrit et remplace par:
    • Si tu testes sur un téléphone physique: L'adresse IP de ton PC (ex: 192.168.1.20)
    • Si tu testes sur l'émulateur: Écris 10.0.2.2 (c'est l'adresse spéciale pour accéder à ton PC)
  4. Pour le port, mets 8000 (ou le port que tu utilises)
  5. Appuie sur Save
✅ C'est bon si: Tu reviens à l'écran principal et les paramètres sont sauvegardés

🌐TEST 2: Tester la connexion à l'API

  1. Sur l'écran principal, appuie sur Test API Connection
  2. Attends 2-3 secondes...
✅ C'est bon si tu vois:
Status: ✅ healthy (en vert)
❌ Si tu vois une erreur:
- Vérifie que le serveur backend est en cours d'exécution
- Vérifie l'adresse IP et le port que tu as configurés
- Essaie de tester depuis ton PC d'abord (curl ou navigateur)

🗄️TEST 3: Tester la connexion à la base de données

  1. Sur l'écran principal, appuie sur Test Database Connection
  2. Attends 2-3 secondes...
✅ C'est bon si tu vois:
Status: ✅ connected (en vert)
Version: [Version de SQL Server]
Server Time: [Heure du serveur]
❌ Si tu vois une erreur:
- Le serveur backend n'a pas pu se connecter à la base de données SQL Server
- Vérifie que SQL Server est en cours d'exécution
- Vérifie les identifiants dans le fichier .env du backend

📝TEST 4: Voir les logs (journaux)

  1. Appuie sur le bouton View Logs
  2. Tu devrais voir une liste de tous les appels API que tu as faits
✅ Tu devrais voir quelque chose comme:
[2026-05-22 14:30:15] API CALL: GET api/health/database
[2026-05-22 14:30:15] API RESPONSE: api/health/database - 200
[2026-05-22 14:30:20] API CALL: GET api/health/api
[2026-05-22 14:30:20] API RESPONSE: api/health/api - 200

Tu peux aussi:

  • Exporter les logs: Appuie sur "Export Logs" pour les envoyer par email ou SMS
  • Effacer les logs: Appuie sur "Clear Logs" pour recommencer

🔄TEST 5: Tester que la configuration se sauvegarde

  1. Configure l'adresse IP et le port (TEST 1)
  2. Ferme complètement l'app (glisse-la vers le haut ou appuie sur "Quitter")
  3. Relance l'app depuis le menu principal de ton téléphone
  4. Appuie sur Settings
✅ C'est bon si: Les paramètres que tu as sauvegardés sont toujours là!
(L'app se souvient de ton IP et ton port même après fermeture)

🔧Aide au dépannage

❌ L'app ne s'installe pas

  • Vérifie que gradlew build a réussi (BUILD SUCCESSFUL)
  • Vérifie que ton téléphone/émulateur est connecté
  • Essaie de brancher ton téléphone en USB avec mode de développement activé

❌ L'API Connection échoue

  • Vérifie que le serveur backend Python s'exécute
  • Teste depuis ton navigateur: http://[IP]:8000/api/health/api
  • Vérifie que tu as sauvegardé les bons paramètres

❌ La Database Connection échoue

  • Vérifie que SQL Server s'exécute
  • Vérifie que le fichier .env du backend a les bons identifiants
  • Teste depuis ton PC: python main.py puis visite http://localhost:8000/api/health/database

💡 Comment trouver l'adresse IP de ton PC

Ouvre le terminal et tape:

ipconfig

Cherche "Adresse IPv4" - ça ressemble à 192.168.x.x
