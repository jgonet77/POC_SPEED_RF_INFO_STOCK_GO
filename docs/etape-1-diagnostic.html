🔍 Étape 1 : Diagnostic du Crash & Setup

Identification et résolution du problème de lancement (~2 heures)

Le Problème

Symptôme : L'application crash au lancement sur téléphone physique

Message d'erreur :
RuntimeException: Unable to start activity... You need to use a Theme.AppCompat theme

Diagnostic

Étape 1 : Consulter les Logs

Utilisation de Android logcat pour voir l'erreur complète :

adb logcat | grep -i "error\|exception"

L'erreur indique que MainActivity n'utilise pas un thème AppCompat compatible.

Étape 2 : Vérifier AndroidManifest.xml

<activity android:name=".MainActivity" android:theme="@android:style/Theme.Material.Light.DarkActionBar"> <intent-filter> <action android:name="android.intent.action.MAIN" /> <category android:name="android.intent.category.LAUNCHER" /> </intent-filter> </activity>

Étape 3 : Analyser la Classe MainActivity

class MainActivity : AppCompatActivity() { // Extends AppCompatActivity mais manifest utilisait Theme.Material ! }

Conclusion du Diagnostic

Root Cause :
• MainActivity extends AppCompatActivity
• Mais AndroidManifest.xml spécifiait @android:style/Theme.Material.Light.DarkActionBar
• AppCompatActivity EXIGE un thème AppCompat

Mismatch : AppCompat class + Material theme = crash au onCreate()

Solution

Fix : Changer le Thème

Remplacer Theme.Material.Light.DarkActionBar par Theme.AppCompat.Light.DarkActionBar
<activity android:name=".MainActivity" android:theme="@style/Theme.AppCompat.Light.DarkActionBar"> <intent-filter> <action android:name="android.intent.action.MAIN" /> <category android:name="android.intent.category.LAUNCHER" /> </intent-filter> </activity>

Pourquoi Cette Solution Fonctionne

• Theme.AppCompat est compatible avec AppCompatActivity
• Fournit les ressources nécessaires (colors, styles, etc.)
• Gère automatiquement les versions Android (backward compatible)
• Support des Material Design colors et patterns
• Compatible avec toutes les versions Android (8+)

L'app s'affiche correctement sur le téléphone physique.
