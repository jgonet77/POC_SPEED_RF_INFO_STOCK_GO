🔄 Étape 5 : State Management & Observers

Gestion complète de l'état avec LiveData et Observer Pattern (~Intégré)

Problème Rencontré

Erreur : IllegalStateException: Cannot invoke setValue on a background thread

Cause : testConnection() exécuté sur daemon thread, tentant de modifier LiveData synchronement

LiveData vs setValue() vs postValue()

setValue()
• Synchrone, thread-safe via Looper
• DOIT être appelé du thread principal
• Mise à jour immédiate

postValue()
• Asynchrone, peut être appelé de n'importe quel thread
• Envoie un message Looper au thread principal
• Mise à jour garantie mais avec délai minimal

Observers Pattern

Enregistrer les Observers

viewModel.connectionStatus .observe(this) { status -> updateConnectionUI(status) } viewModel.waitTime .observe(this) { time -> waitTimeText.text = time } viewModel.errorMessage .observe(this) { error -> if (error.isNotEmpty()) { errorText.text = "Error: $error" } }

Lifecycle-Aware

Important : Les observers sont lifecycle-aware

• Automatiquement arrêtés quand Activity est en pause
• Redémarrés quand Activity reprend
• Nettoyés quand Activity est détruite

→ Pas de fuites mémoire, pas de NullPointerException

State Machine: ConnectionStatus

DISCONNECTED (initial)
↓ user clicks test
CONNECTING (spinner visible)
↓ API responds
CONNECTED ou ERROR
↓ user clicks cancel
DISCONNECTED
fun updateConnectionUI( status: ConnectionStatus ) { when (status) { ConnectionStatus.DISCONNECTED -> { connectionStatusText.text = "Disconnected" connectionSpinner.visibility = View.GONE testApiButton.isEnabled = false } ConnectionStatus.CONNECTING -> { connectionStatusText.text = "Connecting..." connectionSpinner.visibility = View.VISIBLE testApiButton.isEnabled = false } ConnectionStatus.CONNECTED -> { connectionStatusText.text = "Connected ✅" connectionSpinner.visibility = View.GONE testApiButton.isEnabled = true } ConnectionStatus.ERROR -> { connectionStatusText.text = "Connection Failed ❌" testApiButton.isEnabled = false } } }
