    • Click "View Test Logs" → Dialog avec historique tests
    • Affiche: timestamps, API/DB status, response times
    • Clear Logs button → nettoie l'historique

🔍 Détails: Test Connection & Logs

📡 Flux Test Connection:

  1. User clique "Test Connection"
  2. App valide IP & Port (même validation que Save)
  3. Test 1: API Health Check
    • POST request to http://{IP}:{PORT}/health
    • Timeout: 5 secondes
    • Si succès (200) → continue Test 2
    • Si erreur → log erreur + STOP
  4. Test 2: Database Health Check
    • POST request to http://{IP}:{PORT}/db-health
    • Timeout: 5 secondes
    • Si succès (200 + BD responsive) → overall SUCCESS
    • Si erreur → log erreur (BD disconnected, etc.)
  5. Logs Persistence
    • Tous les tests loggés en mémoire (AppLogger)
    • Historique persiste pendant la session
    • User peut voir "View Test Logs" → Dialog
  6. Toast Message
    • Overall result: "✓ All systems OK" ou "✗ Connection failed - see logs"
