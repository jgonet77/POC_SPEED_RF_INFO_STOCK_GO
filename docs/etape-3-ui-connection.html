🎨 Étape 3 : Interface Utilisateur & Connection Status

Création du layout et du ViewModel MVVM (~3 heures)

Objectif

Créer une interface affichant :

✓ État de connexion (Disconnected / Connecting / Connected / Error)
✓ Icône et spinneur animé
✓ IP et Port du serveur
✓ Temps de réponse
✓ Boutons de test API et Database
✓ Gestion d'état complète

1. Créer l'Enum ConnectionStatus

enum class ConnectionStatus { DISCONNECTED, // État initial CONNECTING, // Test en cours CONNECTED, // Test réussi ERROR // Échec }

2. Créer le ViewModel MVVM

HealthViewModel.kt

class HealthViewModel( val context: Context? ) : ViewModel() { private val _connectionStatus = MutableLiveData<ConnectionStatus>( ConnectionStatus.DISCONNECTED ) val connectionStatus: LiveData<ConnectionStatus> = _connectionStatus private val _waitTime = MutableLiveData<String>("") val waitTime: LiveData<String> = _waitTime fun testConnection(apiHost: String = "", apiPort: Int = 0) { _connectionStatus.postValue( ConnectionStatus.CONNECTING ) _waitTime.postValue("Testing...") thread(isDaemon = true) { val startTime = System.currentTimeMillis() repository.checkApiHealth( onSuccess = { response -> val elapsed = System.currentTimeMillis() - startTime _connectionStatus.postValue( ConnectionStatus.CONNECTED ) _waitTime.postValue( "Connected in ${elapsed}ms" ) }, onError = { error -> _connectionStatus.postValue( ConnectionStatus.ERROR ) _waitTime.postValue("") } ) } } }

Points Importants

postValue() au lieu de setValue() car l'appel vient d'un thread daemon

MutableLiveData pour modifications internes

LiveData exposé pour lecture en public (immutable pattern)

4. Connecter les Observers dans MainActivity

viewModel.connectionStatus.observe(this) { status -> when (status) { ConnectionStatus.DISCONNECTED -> { connectionStatusText.text = "Disconnected" connectionSpinner.visibility = View.GONE cancelConnectionButton.visibility = View.GONE testApiButton.isEnabled = false } ConnectionStatus.CONNECTING -> { connectionStatusText.text = "Connecting..." connectionSpinner.visibility = View.VISIBLE cancelConnectionButton.visibility = View.VISIBLE testApiButton.isEnabled = false } ConnectionStatus.CONNECTED -> { connectionStatusText.text = "Connected ✅" connectionSpinner.visibility = View.GONE cancelConnectionButton.visibility = View.GONE testApiButton.isEnabled = true } ConnectionStatus.ERROR -> { connectionStatusText.text = "Connection Failed ❌" connectionSpinner.visibility = View.GONE cancelConnectionButton.visibility = View.VISIBLE testApiButton.isEnabled = false } } } viewModel.waitTime.observe(this) { time -> waitTimeText.text = time }

Architecture MVVM

Model : HealthCheckResponse (data class)

View : MainActivity + activity_main.xml

ViewModel : HealthViewModel (gère état + logique)

Repository : HealthRepository (accès données)

LiveData : Observables réactifs avec lifecycle-aware
