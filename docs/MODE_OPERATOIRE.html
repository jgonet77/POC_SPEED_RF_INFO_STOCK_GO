📋 Mode Opératoire - POC Stock Mobile

Guide complet d'installation, configuration et tests

📍Important

Ce guide explique comment mettre en place le POC sur votre machine et le tester à la fois sur PC (backend) et sur smartphone (frontend).

Durée estimée: 30-45 minutes (première fois)

✅Partie 1: Prérequis

Sur votre PC

Vous devez avoir:

  • ✅ Python 3.10+ installé (vérifiez avec: python --version)
  • ✅ Git installé
  • ✅ Microsoft SQL Server accessible (local ou réseau)
  • ✅ ODBC Driver 17 for SQL Server installé (for pyodbc)

Sur votre smartphone Android

Vous devez avoir:

  • ✅ Android 7.0+ (API 24+)
  • ✅ Connecté au même réseau WiFi que votre PC (important!)

Sur votre PC (pour développement Android)

Optionnel (si vous voulez compiler l'app):

  • Android Studio installé
  • Android SDK 24+
  • Emulateur Android (ou téléphone physique connecté en USB)

NOTE: Pour le POC, vous pouvez aussi juste utiliser une APK pré-compilée (sera fournie séparément après le développement).

🗄️Partie 2: Vérifier la connexion SQL Server

1Connaître vos identifiants SQL Server

Vous avez besoin de:

  • Host: localhost ou IP du serveur (ex: 192.168.1.100)
  • Port: 1433 (défaut)
  • Database: WMS_SPEED
  • Username: sa (ou autre compte)
  • Password: votre password SQL Server

2Tester la connexion avec SQL Server Management Studio

Ouvrez SSMS et essayez de vous connecter avec ces identifiants. Assurez-vous que ça fonctionne avant de continuer.

⚠️ Important pour pyodbc

Vous devez avoir installé ODBC Driver 17 for SQL Server sur votre PC.

Pour Windows: Téléchargez depuis Microsoft ou installez via:

# Windows PowerShell choco install sql-server-odbc-driver

⚙️Partie 3: Mettre en place le Backend (Python)

1Cloner ou télécharger le projet

cd D:\ git clone POC_SPEED_RF_INFO_STOCK cd POC_SPEED_RF_INFO_STOCK

2Créer un environnement virtuel Python

cd backend python -m venv venv .\venv\Scripts\activate # Windows # ou: source venv/bin/activate # Mac/Linux

Après activation, votre terminal doit afficher (venv) au début de chaque ligne.

3Installer les dépendances

pip install -r requirements.txt

4Configurer les variables d'environnement SQL Server

Créez un fichier .env dans le dossier backend/:

SQL_SERVER_HOST=localhost SQL_SERVER_PORT=1433 SQL_SERVER_DB=WMS_SPEED SQL_SERVER_USER=sa SQL_SERVER_PASSWORD=YourPassword123! API_HOST=0.0.0.0 API_PORT=8000

Remplacez les valeurs par vos identifiants SQL Server réels.

⚠️ Sécurité (POC only)

Les variables d'environnement en dur sont OK pour le POC, mais JAMAIS en production! Utilisez un secret manager (Azure Key Vault, etc.)

5Lancer le backend

python main.py

Sortie attendue:

INFO: Started server process [1234] INFO: Uvicorn running on http://0.0.0.0:8000 INFO: Application startup complete

✅ Backend lancé!

Le serveur tourne maintenant sur http://localhost:8000 (sur votre PC)

🧪Partie 4: Tester le Backend depuis votre PC

1Ouvrir un navigateur

Allez à: http://localhost:8000

Vous devriez voir:

{ "message": "Stock API is running" }

2Tester l'endpoint API health

Allez à: http://localhost:8000/api/health/api

Résultat:

{ "status": "healthy", "message": "API is running" }

3Tester l'endpoint Database health

Allez à: http://localhost:8000/api/health/database

Si SQL Server est connecté:

{ "service": "Database Connection Test", "database_status": "connected", "details": { "version": "Microsoft SQL Server 2019 (RTM-CU10)...", "server_time": "2026-05-22 14:30:45.123456" } }

Si erreur de connexion:

{ "detail": "Database connection failed: ..." }

→ Vérifiez vos identifiants SQL Server et relancez le serveur!

✅ Backend fonctionne!

Maintenant, testons depuis le téléphone Android.

📱Partie 5: Préparer l'app Android

Option A: Utiliser l'APK compilée (facile)

1Télécharger l'APK

Une APK pré-compilée sera fournie: stock-app-poc.apk

2Transférer l'APK au téléphone

  • Connectez votre téléphone en USB
  • Copiez l'APK dans: /sdcard/Download/
  • Ou utilisez: adb install stock-app-poc.apk

3Installer l'app

  • Ouvrez le gestionnaire de fichiers du téléphone
  • Allez dans "Téléchargements" (Downloads)
  • Cliquez sur l'APK
  • Acceptez les permissions
  • Installez

Option B: Compiler avec Android Studio (avancé)

1Ouvrir le projet Android dans Android Studio

cd android # Ouvrez le fichier build.gradle dans Android Studio

2Build et déployer

Dans Android Studio: Build → Generate Signed Bundle/APK
Ou directement: Run sur votre téléphone (connecté en USB)

⚙️Partie 6: Configurer l'app Android pour accéder au backend

⚠️ Important: Adresse du backend

L'app Android doit pouvoir accéder au backend Python qui tourne sur votre PC.

L'adresse du backend dépend de votre configuration:

Scénario Adresse du backend dans l'app Conditions
Émulateur Android http://10.0.2.2:8000 10.0.2.2 = adresse spéciale pour l'emulateur qui pointe vers la machine hôte
Téléphone physique sur WiFi http://192.168.X.X:8000 Remplacez X.X par l'adresse IP de votre PC sur le réseau local
Téléphone connecté en USB (adb forward) http://localhost:8000 Après avoir configuré adb port forwarding

1Trouver l'adresse IP de votre PC

Ouvrez PowerShell et tapez:

ipconfig

Cherchez "IPv4 Address" dans votre réseau WiFi (ex: 192.168.1.50)

2Modifier l'adresse du backend dans l'app

Si vous compilez vous-même:

Fichier: android/app/src/main/java/com/example/stockapp/api/ApiClient.kt

object ApiClient { // CHANGER cette URL private const val BASE_URL = "http://192.168.1.50:8000/" // Votre IP PC ... }

📱Partie 7: Tester depuis le téléphone Android

1Lancer l'app Stock

Sur votre téléphone, trouvez l'app "Stock App POC" et ouvrez-la.
Vous verrez deux boutons: "Test API Connection" et "Test Database Connection"

2Tester la connexion API

Cliquez sur "Test API Connection"

Résultat attendu:

  • ✅ healthy (en vert) = Connection au backend réussie
  • ❌ Failed (en rouge) = L'app ne peut pas accéder au backend

Si échoue:

  • Vérifiez que l'adresse IP du backend est correcte
  • Assurez-vous que le téléphone et le PC sont sur le même WiFi
  • Vérifiez le firewall (le port 8000 doit être accessible)

3Tester la connexion Database

Cliquez sur "Test Database Connection"

Résultat attendu:

  • ✅ connected = La DB est accessible (affiche version et heure)
  • ❌ Failed = Problème de connexion DB sur le backend

Si échoue:

  • Vérifiez vos identifiants SQL Server sur le backend
  • Vérifiez que SQL Server tourne
  • Vérifiez les logs du backend (console Python)

✅ POC fonctionne!

Si vous voyez "connected" sur le Database Health test, c'est que votre app Android parle avec SQL Server via le backend! 🎉

🔧Partie 8: Dépannage (Troubleshooting)

Backend ne démarre pas

Erreur: "ModuleNotFoundError: No module named 'fastapi'"

Solution:

pip install -r requirements.txt

Erreur: "Database connection failed"

Solutions:

  • Vérifiez que SQL Server est en train de tourner
  • Testez la connexion avec SQL Server Management Studio
  • Vérifiez le host/port dans les variables d'environnement (.env)
  • Vérifiez que ODBC Driver 17 est installé

App Android ne se connecte pas au backend

Erreur: "Connection Error" dans l'app

Solutions:

  • Vérifiez que le PC et le téléphone sont sur le même WiFi
  • Vérifiez l'adresse IP du backend (utilisez ipconfig sur PC)
  • Testez: ping 192.168.1.50 depuis le téléphone (connecté au WiFi)
  • Vérifiez que le port 8000 n'est pas bloqué par le firewall
  • Ouvrez PowerShell et lancez: netsh advfirewall firewall add rule name="FastAPI" dir=in action=allow protocol=tcp localport=8000

Erreur: "API Error: 500 Database connection failed"

Solution: Le backend s'est connecté, mais la DB ne répond pas. Vérifiez SQL Server.

Logs utiles

Pour déboguer:

  • Regardez la console du backend Python (messages d'erreur)
  • Utilisez Android Studio Logcat pour lire les logs de l'app
  • Tesez les endpoints manuellement: curl http://localhost:8000/api/health/database

App Android prête:

  • ☐ Téléphone et PC sur le même WiFi
  • ☐ Firewall autorise le port 8000

Tests passent:

  • ☐ "Test API Connection" retourne ✅
  • ☐ "Test Database Connection" retourne ✅
  • ☐ Version et heure du serveur affichées

🎉 Vous avez validé la faisabilité technique!

Android ↔ Backend Python ↔ SQL Server
C'est la base du POC. Bravo! 🚀
