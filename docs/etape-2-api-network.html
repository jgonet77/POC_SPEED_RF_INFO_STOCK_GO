🔒 Étape 2 : Intégration API & Network Security

Configuration HTTP et création du client Retrofit (~1.5 heures)

Problème Identifié

Erreur : CLEARTEXT communication to 192.168.1.20 not permitted by network security policy

Cause : Android 9+ bloque le trafic HTTP non chiffré par défaut pour des raisons de sécurité.

Solution : Network Security Config

Créer network_security_config.xml

Fichier : res/xml/network_security_config.xml
<?xml version="1.0" encoding="utf-8"?> <network-security-config> <domain-config cleartextTrafficPermitted="true"> <domain includeSubdomains="true">192.168.1.20</domain> <domain includeSubdomains="true">localhost</domain> <domain includeSubdomains="true">127.0.0.1</domain> </domain-config> <default-config cleartextTrafficPermitted="false" /> </network-security-config>

Référencer dans AndroidManifest.xml

<application ... android:networkSecurityConfig="@xml/network_security_config"> </application>

Intégration Retrofit (HTTP Client)

1. Créer l'interface API

Fichier : api/StockApiService.kt
interface StockApiService { @GET("api/health/database") fun checkDatabaseHealth(): Call<HealthCheckResponse> @GET("api/health/api") fun checkApiHealth(): Call<ApiHealthResponse> }

2. Initialiser le client Retrofit

Fichier : api/ApiClient.kt

Singleton pattern pour éviter les instances multiples
object ApiClient { lateinit var apiService: StockApiService fun init(context: Context) { val baseUrl = ConfigManager(context) .getApiHost() val port = ConfigManager(context) .getApiPort() val retrofit = Retrofit.Builder() .baseUrl("http://$baseUrl:$port/") .addConverterFactory( GsonConverterFactory.create() ) .build() apiService = retrofit.create( StockApiService::class.java ) } }

3. Créer les Data Classes

data class HealthCheckResponse( val service: String, val database_status: String, val details: Map<String, Any> ) data class ApiHealthResponse( val status: String, val message: String )

Gestion des Callbacks Asynchrones

class HealthRepository { fun checkDatabaseHealth( onSuccess: (HealthCheckResponse) -> Unit, onError: (String) -> Unit ) { ApiClient.apiService .checkDatabaseHealth() .enqueue(object : Callback<HealthCheckResponse> { override fun onResponse( call: Call<HealthCheckResponse>, response: Response<HealthCheckResponse> ) { if (response.isSuccessful) { onSuccess(response.body()!!) } else { onError("API Error: ${response.code()}") } } override fun onFailure( call: Call<HealthCheckResponse>, t: Throwable ) { onError("Connection Error: ${t.message}") } }) } }

Points Clés

✅ Cleartext Permission : Strictement limité à 192.168.1.20 en local

⚠️ Sécurité Production : Utiliser HTTPS avec certificats valides

📦 Retrofit : Gère automatiquement threading, sérialisation JSON, retry logic

🔄 Callbacks : Exécutés sur background thread, nécessite postValue() pour LiveData
