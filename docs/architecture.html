Composants Principaux

Frontend (Android Kotlin)

  • MainActivity : Écran principal avec UI de test de connexion
  • HealthViewModel : Gestion d'état MVVM avec LiveData
  • HealthRepository : Abstraction de l'accès API
  • ConfigManager : Stockage des paramètres (IP, port)
  • AppLogger : Journalisation centralisée
  • Retrofit Client : HTTP client typé avec callbacks asynchrones

Backend (Python FastAPI)

  • API Routes : Endpoints REST (/api/health/database, /api/health/api)
  • HealthService : Logique métier de validation de connexion
  • HealthRepository : Accès direct à la base de données SQL Server
  • Pydantic Schemas : Validation et sérialisation des données

Base de Données (SQL Server)

  • Utilisateurs : Authentification et autorisations
  • Stock : Informations de stock par SKU
  • Emplacements : Localisation en entrepôt
  • Connexion : Via pyodbc (driver SQL Server natif)

Flux de Données

1. User Action (UI)
Utilisateur clique sur "Test Connection"
2. ViewModel Action
viewModel.testConnection() → Change LiveData state à CONNECTING
3. Repository Call
repository.checkApiHealth() → Retrofit call asynchrone
4. Network Request
HTTP GET /api/health/api → Server FastAPI
5. Backend Processing
Service valide → Repository exécute requête SQL → Retourne résultat
6. Response Parsing
Retrofit désérialise JSON → HealthCheckResponse
7. LiveData Update
_connectionStatus.postValue(CONNECTED) (thread-safe)
8. UI Refresh
Observer détecte changement → updateConnectionUI() met à jour écran

Configuration & Sécurité

Network Security Config (Android)

Fichier : network_security_config.xml

Permet le trafic HTTP non chiffré UNIQUEMENT pour :
  • 192.168.1.20 (serveur local)
  • localhost / 127.0.0.1
Important : HTTP est bloqué par défaut en Android 9+. Cette configuration est strictement pour le POC en environnement local.

Configuration Manager (SharedPreferences)

Stocke localement :
  • apiHost (IP du serveur)
  • apiPort (port de l'API)
Permet à l'utilisateur de modifier ces valeurs via Settings Activity.

Thread Safety & Asynchronisme

Android : LiveData + Coroutines

  • setValue() : Met à jour depuis le thread principal
  • postValue() : Met à jour depuis un thread quelconque (thread-safe)
  • Retrofit callbacks : Exécutés sur background thread
  • Observers : Appelés automatiquement sur main thread

Backend : pyodbc + FastAPI

  • FastAPI : Framework async natif en Python
  • pyodbc : Connexion synchrone à SQL Server
  • Pas de gestion thread complexe pour ce POC
  • Connection pooling possible pour montée en charge
